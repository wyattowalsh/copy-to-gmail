# Gmail Draft Sync Goal

Add optional Gmail integration to Copy to Gmail so users can connect one Gmail account, load and sync Gmail drafts, open synced drafts in Gmail, and manage local templates, variables, signatures, and version snapshots without adding a database.

The shared understanding is captured in `facts.md`. The execution path is captured in `plan.md`.

Done means the existing local-only compose, preview, validation, copy, and JSON export/import workflows still work without login, while connected users can safely create, load, autosync, conflict-resolve, version, and open Gmail drafts with clear docs about scopes and local storage.
