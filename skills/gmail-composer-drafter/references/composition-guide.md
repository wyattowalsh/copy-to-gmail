## Drafting Defaults

- Start with the user's goal, audience, and relationship when provided.
- Prefer one primary call to action.
- Keep paragraphs short enough to scan in Gmail compose.
- Use bullets only when they reduce cognitive load.
- Provide labeled link text plus the URL when the user supplied links.
- Keep claims grounded in user-provided facts.
- Use preview text to clarify the value or ask without repeating the subject exactly.

## Tone Controls

| Tone    | Use When                                | Style                                     |
| ------- | --------------------------------------- | ----------------------------------------- |
| Direct  | Operational updates, clear asks         | Short sentences, explicit CTA             |
| Warm    | Relationship maintenance, follow-ups    | Human opening, low-pressure CTA           |
| Formal  | Legal, executive, external stakeholders | Precise wording, fewer contractions       |
| Launch  | Announcements, campaigns                | Benefit-first opening, concrete next step |
| Apology | Repair or delay updates                 | Own the issue, state fix, avoid excuses   |

## Editor Handoff

Use **Editor Notes** for anything about the local tool:

- Suggested heading hierarchy.
- Whether to make a phrase bold or linked.
- Whether a CTA should be a paragraph link or button.
- Reminder to click **Copy for Gmail** and paste with Gmail Plain text mode off.

Do not put tool instructions inside **Email Body**.

## Refusal Boundaries

Refuse or redirect when the user asks to:

- send the email,
- access Gmail messages,
- create Gmail filters or labels,
- triage inboxes,
- generate raw HTML as the primary artifact,
- invent unverified business or personal facts.

## Quality Checks

Before returning a draft, check that:

- the email has a single clear audience,
- the call to action is visible without being pushy,
- all dates, links, prices, claims, and names came from the user,
- the body can be pasted into the editor without markdown-only syntax,
- editor-only instructions appear only in **Editor Notes**.
